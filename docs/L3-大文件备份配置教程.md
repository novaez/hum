# L3 教程 2：配置大文件备份（可选）

> ⚠️ 这套方案用 **GitHub private repo** 存 PDF、PPT 等不走 iCloud 的大文件。Repo 私有，不会泄露。

## 你会得到什么

- vault 根目录下 `Attachments.nosync/` 专门放大文件
- 目录名 `.nosync` 后缀 + 里面放空 `.nosync` 文件，iCloud 自动忽略
- **往里一扔新文件就自动 commit + push 到 GitHub**（秒级）
- 换电脑 clone 一份恢复

## 步骤

### 1. 建 GitHub private repo

- github.com → `+` → `New repository`
- Name：`attachments`（或你喜欢的）
- **Visibility: Private**
- 勾 `Add a README file` → `Create repository`

### 2. 配 SSH key

一次终身无忧，避开 PAT 过期维护。终端跑：

```bash
# 2.1 生成 key（已有 ~/.ssh/id_ed25519 可跳过）
ssh-keygen -t ed25519 -C "你的邮箱"
# 一路回车，passphrase 可留空

# 2.2 加到 macOS Keychain（让 Automator 能无密码用）
ssh-add --apple-use-keychain ~/.ssh/id_ed25519

# 2.3 配 ~/.ssh/config
cat >> ~/.ssh/config << 'EOF'

Host github.com
  HostName github.com
  User git
  AddKeysToAgent yes
  UseKeychain yes
  IdentityFile ~/.ssh/id_ed25519
EOF

# 2.4 用教程 1 的 PAT 自动上传公钥到 GitHub（省去 UI 粘贴步骤）
read -s -p "粘贴你的 PAT（屏幕不会显示，粘完回车）: " PAT; echo
curl -sL -X POST \
  -H "Authorization: Bearer $PAT" \
  -H "Accept: application/vnd.github+json" \
  https://api.github.com/user/keys \
  -d "{\"title\":\"$(hostname)\",\"key\":\"$(cat ~/.ssh/id_ed25519.pub)\"}"
unset PAT
```

API 返回带 `"id": <数字>` 和 `"verified": true` 就是成功；如果返回 `"message": "..."` 就是报错（常见：PAT 没勾 `write:public_key` scope）。

**验证**：

```bash
ssh -T git@github.com
# 成功输出 "Hi <你的用户名>! You've successfully authenticated..."
```

### 3. Vault 里建 `Attachments.nosync/` 并 init git

```bash
cd ~/Library/Mobile\ Documents/iCloud~md~obsidian/Documents/你的vault名字
mkdir Attachments.nosync && cd Attachments.nosync
touch .nosync   # 双保险

git init
git branch -m main
git remote add origin git@github.com:你的用户名/attachments.git
git pull origin main --allow-unrelated-histories
# 可能弹 merge commit 编辑器：保持默认信息，:wq 保存退出
git add -A
git commit -m "init"
git push -u origin main
```

### 4. 写 auto-sync 脚本

把下面内容存到 `~/sync_attachments.sh`（记得替换 `TARGET_DIR`）：

```bash
#!/bin/bash
LOG_FILE="$HOME/sync_attachments.log"
TARGET_DIR="/Users/你的用户名/Library/Mobile Documents/iCloud~md~obsidian/Documents/你的vault名字/Attachments.nosync"

# 自清理：log 超 50KB 就截到后半
if [ -f "$LOG_FILE" ] && [ "$(stat -f%z "$LOG_FILE" 2>/dev/null)" -gt 50000 ]; then
  tail -c 25000 "$LOG_FILE" > "$LOG_FILE.tmp" && mv "$LOG_FILE.tmp" "$LOG_FILE"
fi

exec >> "$LOG_FILE" 2>&1
echo "---- $(date) ----"

cd "$TARGET_DIR" || { echo "ERROR: cd failed"; exit 1; }
export PATH="/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin:$PATH"

git add -A
if [[ -n $(git status -s) ]]; then
  git commit -m "Auto-sync: $(date +'%Y-%m-%d %H:%M:%S')"
  git push origin main && echo "sync done"
else
  echo "no changes"
fi
```

加可执行权限：

```bash
chmod +x ~/sync_attachments.sh
```

### 5. Automator Folder Action 绑定

- 打开 Automator.app
- `File → New` → 选 **Folder Action**
- 顶部 "Folder Action receives files and folders added to" 下拉 → `Other...` → 选 `Attachments.nosync/`
- 左边搜 `Run Shell Script`，拖到右边
- Shell 选 `/bin/bash`
- 命令填：`/Users/你的用户名/sync_attachments.sh`
- `File → Save` → 起名 "Sync Attachments"

### 6. 授权"完全磁盘访问"（如果 Folder Action 不触发）

System Settings → Privacy & Security → Full Disk Access → 加入 `Automator` 和 `FolderActionsDispatcher`

## 验证

- 往 `Attachments.nosync/` 扔测试文件
- 几秒内 `tail ~/sync_attachments.log` 看到 `sync done`
- github.com 看 private repo，有新 `Auto-sync:` commit

## 排错

- **没触发**：去 Full Disk Access 加 Automator
- **log 里 SSH 错**：重测 `ssh -T git@github.com`；passphrase 确认进了 Keychain
- **log 里 git push 错**：repo 权限、SSH key 是否添加到 GitHub
- **API 返回 `"message": "Resource not accessible..."`**：PAT 没勾 `write:public_key` scope。回 GitHub → Developer settings 编辑 PAT，补上这个 scope 再重跑
- **`.DS_Store` 会被跟踪吗**：macOS Folder Action 对 `.DS_Store` 变化做了过滤，所以不会触发 sync；但 `git add -A` 会把它加进 commit。如果嫌噪声，vault 根的 `.gitignore`（或 repo 的 `.gitignore`）加一行 `.DS_Store` 即可

## 补充

- **换电脑恢复**：`git clone git@github.com:你的用户名/attachments.git`
- **手动触发同步**：`bash ~/sync_attachments.sh`

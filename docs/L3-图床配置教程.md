# L3 教程 1：配置 GitHub 图床（可选）

> ⚠️ **隐私提示**：这套方案把图片传到 **public GitHub repo**，全世界能看到（URL 不公开传播，但理论上可被爬取）。涉密/涉私截图别贴。私密方案待后续更新。

## 你会得到什么

- 粘贴截图到 Obsidian 自动上传，本地不留图片
- 走 jsdelivr CDN，国内外都快
- markdown 里存的是公开 URL，换设备/分享直接能加载

## 步骤

### 1. 装 PicList（一个把图片推 GitHub 的独立 app）

两选一：

- 终端跑 `brew install --cask piclist`（需要先装 Homebrew，见 https://brew.sh）
- 或者去 https://piclist.cn 下 dmg 安装

系统要求：macOS ≥ 12

### 2. 建一个 GitHub public repo 作图床

- 没 GitHub 账号先注册：github.com
- 登录 → 右上角 `+` → `New repository`
- Repository name 起个短名，比如 `my-imgs`
- **Visibility 选 Public**（必须，jsdelivr 进不去 private）
- 勾 `Add a README file` → `Create repository`

### 3. 建一个 Personal Access Token（PAT）

- github.com 右上头像 → `Settings`
- 左侧拉到底 → `Developer settings` → `Personal access tokens` → `Tokens (classic)` → `Generate new token (classic)`
- Note 填任意名字（比如 `piclist`）
- Expiration 选 `No expiration`
- 勾选 `repo` 大类（下面几项会自动全勾）
- 再勾选 `admin:public_key` → 里面的 `write:public_key`（教程 2 上传 SSH 公钥要用，一次搞定免得后面再来一趟）
- 拉到底 `Generate token` → **立即复制**（关闭页面就看不到了）

### 4. PicList 里配 GitHub 图床

打开 PicList，左边栏 `PicBed` 展开 → 点 `GitHub`，填：

| 字段 | 填什么 |
|---|---|
| Config Name | 任意，比如 `GitHub Image Bed` |
| Set Repo Name | `你的用户名/repo 名`（比如 `alice/my-imgs`） |
| Set Branch | `main` |
| Set Token | 粘贴第 3 步复制的 PAT |
| Set Path | 留空（或填 `img/` 分目录） |
| Set Web Path | 留空 |
| Set Custom URL | `https://cdn.jsdelivr.net/gh/你的用户名/repo 名@main` |

保存后右键这个配置 → 设为默认图床。

**测试**：左边栏 `Upload` → 拖张图进去，看下方是否显示 `https://cdn.jsdelivr.net/gh/...` 的链接。能出 URL 才算通。

### 5. 启用 PicList 的 Upload API 服务（给 Obsidian 插件调用）

左边栏 `Settings` → 顶部标签 `Advanced` → 往下翻到 `Server Settings` → 点 `Upload API Service Settings`：

- 打开 `Enable Upload API Service` 开关
- Listen Port 保持 `36677`
- Authentication Key 留空（无需认证）
- 点 `Confirm`

### 6. Obsidian 里配 image-auto-upload 插件

Obsidian → `Settings` → Community plugins → 找到 `Image auto upload Plugin`（L2 已装）→ 点设置：

- Uploader：`PicGo`
- Upload server：`http://127.0.0.1:36677/upload`
- Delete server：`http://127.0.0.1:36677/delete`
- 打开「启用剪贴板自动上传」

## 验证

回 Obsidian，粘贴一张截图到任意笔记。成功的表现：

- 插件提示上传成功
- markdown 里插入的是 `https://cdn.jsdelivr.net/gh/...` 链接
- 本地文件系统没留图片文件
- 推回 GitHub repo 打开看，图片在里面
